# CLAUDE.md — mono.create 社内連絡アプリ 開発ガイド

このリポジトリは単一ファイル構成の社内連絡PWA(`index.html`)です。
UIの追加・変更を行うときは、必ず以下のデザインルールに従ってください。

## デザインの基本方針: Mobbin を参考にする

**UIを新規作成・変更するときは、必ず [Mobbin](https://mobbin.com)(実在アプリのUIスクリーンショット/デザインパターン集)に掲載されているような、実績のあるプロダクトのデザインパターンを参考にしてデザインを組むこと。**
自己流のレイアウトを一から発明せず、ユーザーが既に使い慣れている定番アプリのパターンに寄せる。

### ワークフロー

1. **参考指定がある場合(最優先)**
   ユーザーが Mobbin のスクリーンショット・URL・アプリ名を提示した場合は、そのデザインを最優先の参照元として忠実に再現する(配色・フォントは本アプリのトークンに置き換える)。
2. **参考指定がない場合**
   作ろうとしている画面の種類に対して、Mobbin に掲載されている定番アプリの該当パターンを自分で選び、「◯◯のパターンを参考にした」と明示した上で実装する。
3. **迷ったら聞く**
   複数の有力パターンがあり方向性が大きく変わる場合は、候補アプリ名を挙げてユーザーに確認する。

### 画面タイプ別の参考パターン(デフォルト)

| 画面タイプ | 参考にするアプリ(Mobbin掲載の定番) |
|---|---|
| リスト/テーブル(営業リスト等) | Notion、Linear、Airtable |
| カレンダー/タイムライン | Notion Calendar、Google Calendar、Fantastical |
| チャット/連絡・通知 | Slack、LINE WORKS、Discord |
| ダッシュボード/集計 | Stripe、Linear Insights |
| フォーム/入力モーダル | Linear の Issue 作成、Things |
| 設定画面 | iOS 設定、Slack Preferences |
| 検索/フィルタ | Airbnb、Notion のクイック検索 |
| 空状態(Empty State) | Linear、Notion(イラスト+一言+主要アクション1つ) |
| オンボーディング/ヘルプ | Duolingo、Notion |

### Mobbin 流パターンを適用する際の共通原則

- **情報階層を先に決める**: 1画面1目的。主要アクションは1つだけ強調(`--purple`)、残りはセカンダリ。
- **カード+余白ベース**: 定番アプリ同様、境界線より余白とカード(`--card`, `--shadow-sm`)で区切る。
- **既存トークンを必ず使う**: 色は `:root` の CSS変数(`--bg`, `--surface`, `--purple`, `--text`, `--t2` など)、角丸は `--r` / `--rs`、影は `--shadow*` を使用。新しい生値(hex直書き)を増やさない。
- **モバイルファースト**: このアプリはPWAとしてスマホで使われる。Mobbin の iOS/Android パターンを基準にし、タップ領域は最低44px。
- **状態を省略しない**: ローディング・空状態・エラーの3状態を定番アプリのパターンで必ず用意する。
- **日本語UIの密度**: 日本語は情報密度が高くなりやすいので、参考パターンより行間・余白をやや広めに取る。

## 技術メモ

- 全コードは `index.html` 1ファイル(HTML/CSS/JSインライン)。ビルド工程なし。
- デザイントークンは `index.html` 冒頭の `:root` に定義済み。
- フォント: `-apple-system, 'Hiragino Sans', 'Yu Gothic', ...`(システムフォント)。
- PWA: `manifest.json` + `sw.js`。`theme-color` は黒。
